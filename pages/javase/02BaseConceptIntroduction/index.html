# HelloWorld

// 记事本编写，保存为 HelloWorld.java
public class HelloWorld{
    public static void main(String[] args){
        System.out.println("HelloWorld");
    }
}

HelloWorld 的分析：

  • 源代码文件以 .java 为扩展名；
  • public static void main(String[] args){} 称之为 main 方法，其花括号中为方法体，main 方法是程序执行入口（目前可简单理解为，没有 main 方法，即使程序语法和逻辑正确，程序也无法执行相应操作）；
  • 代码严格区分大小写（即 A 和 a 是不同的），每条语句以英文分号 ; 结束，花括号后省略；
  • 源代码中有且仅有一个 public class （公共类）且和源代码文件名保持一致。可以存在与之『平级』的 class ，其个数、名字不限。使用 javac 公共类名 会同时将其他 class 类一起编译。
// 记事本编写，保存为 HelloWorld.java
public class HelloWorld{
    public static void main(String[] args){
        System.out.println("HelloWorld");
    }
}
class helloWorld{
    public static void main(String[] args){
        System.out.println("helloWorld");
    }
}

# HelloWorld 的一生

HelloWorld.java 文件 >>（ javac ）编译器编译 >> HelloWorld.class 字节码文件 >>（ java ) JVM 执行 >> 结果

这就是 HelloWorld 的一生，同时也是所有 Java 程序的一生，也叫 Java 程序的执行流程。

# 浅聊 JVM

JVM（Java Virtual Machine） 即 Java 虚拟机，可以将其理解为一台运行在你的计算机中的计算机。

正是有了这台虚拟的计算机，为 Java 提供了跨平台性。无论代码是在 Windows 还是 macOS，亦或者是 Linux 平台编译，都能运行在其他同样安装了 JVM 的操作系统上。

这就是 Java 的跨平台性，即『一次编写，处处执行』。

# 关键字和保留字

在 HelloWorld.java 中，我们注意到， class 这个单词被我们称之为类。能否使用其他单词或者拼音去取代它？答案是否定的。因为 class 在 Java 中已经被定死了，用它作为『前缀』去标记的东西才叫类。也就是说：

  • 关键字 被 Java 语言赋予了特殊含义，用做专门用途的字符串（单词）。
  • 保留字 现有 Java 版本尚未使用，但以后版本可能会作为关键字使用。

# 为什么不是 helloWorld

我们在创建 HelloWorld.java 文件时，为什么不能是 helloWorld.java 或者其他拼写方式呢？这就涉及到命名规则和命名规范的问题，它不仅涉及到类名，还涉及到后面我们所有需要命名的地方。

命名规则 必须遵守，不遵守将会使程序无法正常工作。

命名规范 目的是便于区分命名的对象，方便程序员理解阅读，不唯一。

# 命名规则

  1. 由数字、英文字母、下划线或 $ 组成；
  2. 不能独立使用关键字和保留字，但能包含；
  3. 严格区分大小写，长度无限制，不能有空格。

# 命名规范

  • 包名：多单词组成时所有字母小写，推荐域名反转，如 com.google.test ；
  • 类名、接口名：多单词组成时，大驼峰（首字母大写）；
  • 变量名、方法名：多单词组成时，小驼峰（首字母小写）；
  • 常量名：全大写，多单词组成时，下划线做分隔。

上述名称后面会有介绍，无需死记硬背，用多了就能记住。

# 拆开 HelloWorld

试一试 HelloWorld.java 中，修改为 System.out.println("Hello\nWorld"); 会发生什么？

转义字符 当输入某些特定的字符时，会被赋予特殊含义。

# 常见转义字符

\t 制表符；\n 换行；\\ 反斜杠；\'' 双引号；\' 单引号

# 与代码无关又与代码有关

当我们想在代码中给自己一些提示时，如果不加任何符号，代码将会发生错误。而如果插入注释，编译器将会忽略我们这些与代码『无关』的文字。

# 注释

  • 单行注释 //
  • 多行注释 /* */
  • 文档注释 /** */

# Java 代码规范

  • 类、方法的注释，以 javadoc（文档注释） 形式书写；
  • 使用 tab 缩进（四个空格）；
  • 符号两边各加一个空格；
  • 编码使用 UTF-8；
  • 代码编写使用行尾风格（推荐）或次行风格；
  • 更多详见阿里巴巴代码规范。

此处仅作整理，目前了解即可，本教程的代码都遵循这些规则规范。
