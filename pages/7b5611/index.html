泛型

# 什么是泛型

回想集合框架，当我们存入元素时，无论是 int 、 String 还是其他数据类型，最终都会成为一个 Object 对象。

将对象从中取出时，还需要我们手动进行一次强制类型转换。当存在与预想中不同的类型时，还会发生类型转换错误。

泛型的出现，解决了上述的问题。以 ArrayList 为例，新的书写方式 ArrayList<E> al = new ArrayList<E>(); ，其中 T 可以是任一 Java 类型，但只能是引用类型。

ArrayList<String> strL = new ArrayList<String>();
ArrayList<Integer> intL = new ArrayList<Integer>();
ArrayList<Person> perL = new ArrayList<Person>();

此时，只能存入指定类型的元素。也就是说，如果不指定 T 则默认为 Object 。

总结 泛型作用就是等待确定其类型。

# 自定义泛型类

通过已存在的类为例，改造为泛型类。

public class Book {
    private String bookName;
    private String author;
    public Pair(String bookName, String author) {
        this.bookName = bookName;
        this.author = author;
    }
    public String getBookName() {
        return bookName;
    }
    public String getAuthor() {
        return author;
    }
}

将上述改造为泛型类：

public class Book<T> {
    private T bookName;
    private T author;
    public Pair(T bookName, T author) {
        this.bookName = bookName;
        this.author = author;
    }
    public T getBookName() {
        return bookName;
    }
    public T getAuthor() {
        return author;
    }
}

完成，此时我们使用该类时，在 <T> 中指定了类型后，所有的 T 将转为指定类型。

// 编译错误
public static Book<T> read(T bookName, T author) {
    return new Book<T>(bookName, author);
}

静态方法的返回类型和方法参数无法使用泛型，我们只需将其改为泛型方法即可。

public static <K> Book<K> read(T bookName, T author) {
    return new Book<K>(bookName, author);
}

泛型的类型的数量并不局限于一个，还可以将其改造为 <T,E,K> 甚至更多， Map<K,V> 就是一个官方典例。

# 泛型小总结

  • 不能是基本类型，例如： int ；
  • 不能获取带泛型类型的 Class ，例如： Pair<String>.class ；
  • 不能判断带泛型类型的类型，例如： x instanceof Pair<String> ；
  • 不能实例化 T 类型，例如： new T() 。

泛型方法要防止重复定义方法，例如： public boolean equals(T obj) ；

子类可以获取父类的泛型类型 <T> 。
